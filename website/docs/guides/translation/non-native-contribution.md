# 5. 비모국어 기여 가이드

기여자의 모국어가 원문 언어와 다를 때 사용할 수 있는 실무 규칙입니다.

## 작성 규칙

1. 원문 문서의 의미를 먼저 고정합니다.
2. 문장은 짧고 직접적으로 작성합니다.
3. API명, 옵션명, 제품명은 원문 표기를 유지합니다.
4. 코드 블록과 명령어는 번역하지 않습니다.

## 리뷰 규칙

1. 기술 정확성 리뷰와 문장 자연스러움 리뷰를 분리합니다.
2. 기술 정확성은 도메인 리뷰어가 확인합니다.
3. 문장 자연스러움은 원문 언어 리뷰어가 확인합니다.

## PR 작성 팁

- 어떤 원문 문서를 기준으로 번역했는지 링크를 명시합니다.
- 번역에서 일부 용어를 원문 그대로 유지했다면 이유를 적습니다.
- scope를 명확히 적습니다.
  - 예: `guides/configuration.md ko 번역 업데이트`

## 권장 체크리스트

- 원문과 번역의 문단 구조가 동일하다.
- 용어 표기 규칙이 문서 전체에서 일관된다.
- locale 페이지에서 링크/제목이 정상적으로 보인다.
